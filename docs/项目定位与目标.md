# 项目定位与目标

> 本文档记录"人类知识地基计划"经过多轮实践后，对自身目标的诚实调整。
> 它不是放弃最初的愿景，而是把愿景拆成可以落地的层次。

## 缘起：一次诚实的调整

原始愿景：**用 AI 对全人类知识体系的隐性地基，做一次系统性的全盘扫描。**

但经过多轮实践（见 `methodology/` 与 `手记/`），我们诚实地发现：**靠"脑暴 + 与大模型对聊"，想从零挖出一个震撼性的全新地基，非常难。** 我们反复落在一些"一旦说清楚、专家其实早就大致知道"的东西上（IID、DNA 蓝图、训练目标 vs 内在目的……）。

这不是失败，这是一个重要发现。于是我们把目标结构调整如下。

## 一、对难度的诚实认识

* 真正深藏的地基，通常由 **"领域内的人 + 长期撞到的异常"** 磨出来（开普勒盯火星轨道误差、马歇尔吞下幽门螺杆菌培养液）。
* **一个要命的悖论**：长期暴露 → 司空见惯 → 视而不见。**最有资格发现异常的人，恰恰最容易对异常麻木。**
* 所以"发现异常 / 看见线索"本身就是一个难点，也是一个值得专门研究的题目。
* **解药之一，正是 `手记/01_儿童般的追问.md` 里那份"儿童般的追问"**——用没被驯化的好奇，去对抗被经验磨出来的盲视。这份手记不是抒情，它是本项目对抗"司空见惯"的核心方法论武器。

## 二、主线目标（现实可行）

> **辅助"已经闻到味道的内行"，把他那点说不清的直觉，磨成清晰、可验证、画得出边界的命题。**

我们不擅长"从零发现"，但很擅长"把模糊直觉磨清楚"。今晚反复做成功：组合能力 vs 取用部分、整体 = 部分 + 关系、写不尽 ≠ 大于、工艺 ≠ 意志。

## 三、次级目标 A：培养"地基审查意识"

把探针变成**可教、可练、可检验的"招式"**，而不是"要敢于质疑"这类鸡汤。

已具雏形的招式：
* **条件与边界**：这个假设成立的条件是什么？条件今天还在不在？
* **基底决定论**：这个"不能"，是机制层不能，还是涌现之后也不能？
* **概念滑动 / 双标**：这个词，是不是在两个意思之间偷偷换脚？
* **证据闸门**：别把裂纹夸成地震。

**防陷阱**：必须给"具体动作"，绝不给"态度口号"。这是我们和鸡汤的唯一区别。

## 四、次级目标 B：做"省力的辅助工具"（顺应人性）

**人性动力学**：主动学习是高功耗行为（专注时大脑约 35-40 瓦，安静时约 20 多瓦），不能指望所有人都主动学习。

* **期望**：几乎人人都想自己获得突破、获得发现。
* **现状**：很难做到。
* **解法**：不求替他做，求**帮他做、辅助他做**，让他更轻松、更容易。

**产品铁律**：工具最好嵌进用户"本来就在做的事"里，在后台帮他留意异常，而不是要求他专门坐下来"学习"。

形象地说，做一个 **"假设的拼写检查器"**：用户正常写代码 / 读论文 / 做研究时，它在旁边默默标注——"这里你默认了一个前提，要不要看一眼它的边界？" 低启动成本、低功耗、顺着人性走。

## 五、引擎方向：把方法交给机器去跑

迄今为止主要是"手工"，机器的速度优势没被发挥。下一步：把探针与原则做成**机器可执行的扫描流程**。

* **机器的长处 = 覆盖面（广撒网）**：速度快，能扫海量候选，提高 recall。
* **机器的短处 = 假阳性高**：它会把大量普通裂纹也报成地震（见 `04_证伪协议.md`）。
* **分工**：**机器广撒网（提高撞见概率），人 + 证据闸门做精筛（保精度）。**
* 哪怕扫不出惊天动地的，把"撞见概率"从万分之一提到百分之一，本身就是大突破。

## 目标之间的关系

```
愿景：对全人类知识地基做系统性扫描
 ├─ 主线：辅助内行，把直觉磨成可验证命题
 ├─ 次级 A：培养"地基审查意识"（把探针变成可教招式）
 ├─ 次级 B：做省力辅助工具（顺应人性，不替他做、帮他做）
 └─ 引擎：机器化扫描（广撒网）+ 证据闸门（精筛）
```

**共同的底层信念：**

> 与其赌自己中一次彩票，不如让一万个人，都拥有挖矿的眼睛和一把省力的铲子。

## 待挖的研究线索

* **"异常"为何会被视而不见？如何系统地对抗"习惯化盲视"？**（关键词：习惯化、惯性盲区、儿童般的追问）——这条线索本身，可能就是一个值得挖的好题目。
