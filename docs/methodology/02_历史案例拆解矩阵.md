# 历史案例拆解矩阵 (Historical Case Matrix)

为了训练我们的 AI 探针，并向外界证明“隐性前提显性化”的规律性，我们需要建立一个历史案例库。

## 统一拆解模板（8问）

对每一个历史上的“范式转移”案例，我们不讲宏大故事，只回答以下 8 个结构化问题：

1. **当时默认了什么？**（找到那个隐性前提）
2. **为什么这个默认在当时很合理？**（找出当年的资源限制或效率考量）
3. **它支撑了什么理论、工具或制度？**（评估其影响面）
4. **哪些异常长期解释不了？**（现实冲突的爆发点）
5. **当时做了哪些局部修补？**（观察“补丁坍塌”的过程）
6. **谁把这个前提显性化了？**（破局者是谁）
7. **前提改写后打开了什么新空间？**（重构后的巨大价值）
8. **它属于哪种“地基裂缝类型”？**（归类到我们的探针系统中）

## 待拆解的初始案例库 (Top 10)

我们将优先对以下 10 个案例进行结构化拆解：

**物理与数学类：**
1. 欧几里得平行公设 -> 非欧几何（黎曼几何）
2. 绝对时间/绝对空间 -> 狭义/广义相对论
3. 静态宇宙直觉 -> 宇宙膨胀（哈勃）

**生命与医学类：**
4. 物种不变论 -> 进化论（达尔文）
5. 生命需要“活力” -> 有机化学的诞生（尿素合成）
6. 胃溃疡主要来自压力/胃酸 -> 幽门螺杆菌的发现

**社会与认知类：**
7. 理性经济人假设 -> 行为经济学（卡尼曼）
8. 智力是单一维度的 -> 多元智能理论

**工程与技术类（重点）：**
9. 机械防卡键妥协 -> QWERTY 键盘布局的全球垄断
10. 省胶片成本妥协 -> 电影 24 帧的“艺术感”幻觉
11. UI必须命令式操作DOM -> 声明式状态映射（React/Vue）
12. 智能必须基于严密逻辑规则（专家系统） -> 概率与涌现（深度学习）
