# 清朗知识翻译计划：反黑话、反误导、反不必要认知负担

## 1. 项目起因

这个想法来自一次关于脑科学术语的讨论。

很多困惑并不是学习者理解能力差，而是知识传播链条本身出了问题：翻译不准确、术语反直觉、学术表达故作高深、科普文本用“黑话”包装简单机制。结果是，原本可以直接理解的概念，被表述方式人为增加了几倍、几十倍甚至上百倍的认知负担。

在信息不发达的时代，翻译、整理、重写和纠错的成本很高，人们只能被动忍受这些糟糕表达。但在 AI 已经大幅降低信息处理成本的今天，我们不应该继续把大量精力浪费在“破解黑话”上。

这个计划的核心动机是：

> 当 AI 让信息处理成本大幅下降时，人类不应再被糟糕翻译、学术黑话和故作高深的表述长期折磨。我们需要一个 AI 翻译层，把知识从“黑话版本”还原成“清晰版本”。

## 2. 核心问题

本计划关注的不是知识本身是否深奥，而是表达方式是否制造了不必要的困难。

需要处理的问题包括：

- **误导性翻译**：字面翻译正确，但会把学习者带向错误理解。
- **学术黑话**：用复杂术语堆砌简单机制，让人误以为概念本身极其神秘。
- **术语反话**：术语名称和实际机制方向相反，导致初学者长期误判。
- **科普歪话**：为了显得高级或故事化，把机制讲成拟人化、宗教化、玄学化的叙述。
- **概念封装过厚**：一个词背后包含多个机制，但文本不拆开说明，导致学习者只能死记词语。

这类问题会造成一种隐性浪费：

> 学习者以为自己在学习知识，其实大量时间消耗在翻译污染、表达污染和术语污染上。

## 3. 核心原则

### 3.1 不降低知识深度，只降低无意义难度

清晰表达不是浅薄化。真正的目标不是把复杂知识讲成鸡汤，而是把复杂性放回它真正存在的地方。

如果一个概念确实复杂，就清楚拆开它的结构；如果只是术语造成复杂错觉，就把术语外壳剥掉。

### 3.2 把“术语”翻译成“机制”

术语不是终点，机制才是终点。

每遇到一个黑话术语，都要追问：

- 它实际指向什么过程？
- 它包含哪些组成部分？
- 它在系统里起什么作用？
- 换成计算机、工程、硬件或日常经验类比后，是否更容易理解？

### 3.3 保留专业性，但拒绝装神弄鬼

专业术语可以保留，但必须配套一个清晰版本。

允许专家继续使用专家语言，但必须增加一个面向普通学习者和跨学科读者的“翻译层”。

### 3.4 先提取规则，再让 AI 批量执行

这个计划不能变成人类挽起裤脚、亲自下地一条条改写术语的苦力工程。

真正要做的是提取方法、规则和判断标准，让 AI 执行大规模翻译、拆解、纠偏和重写。

人类负责：

- 制定原则
- 设计样例
- 审查质量
- 修正规则
- 判断边界

AI 负责：

- 批量识别黑话
- 生成清晰解释
- 给出多个类比版本
- 标注可能误导点
- 形成可复用的“清朗译文”

## 4. 未来可能形态

### 4.1 黑话翻译器

输入一个术语或一段学术文本，输出：

- 原始表达
- 直译问题
- 容易误解的地方
- 清晰机制版
- 工程类比版
- 初学者版
- 保留专业术语的严谨版

### 4.2 术语污染评级

为一个术语打“认知负担分”：

- 是否反直觉
- 是否误导方向
- 是否过度拟人化
- 是否用词过度抽象
- 是否缺少机制拆解

### 4.3 学科清朗词典

从脑科学、心理学、AI、物理学等领域开始，建立一批“黑话 -> 清晰机制”的对照表。

每个条目包括：

- 原术语
- 常见误解
- 更好的解释
- 最小机制图
- 适合初学者的类比
- 适合专业读者的边界说明

### 4.4 AI 批量重写工作流

最终目标不是手工写几篇科普文，而是建立一套可复用工作流：

```text
原始学术文本
  -> 黑话识别
  -> 术语拆解
  -> 机制还原
  -> 类比生成
  -> 误解风险标注
  -> 清朗版本输出
  -> 人类抽检与规则修正
```

## 5. 与人类知识地基计划的关系

“人类知识地基计划”主要处理知识背后的隐含假设。

“清朗知识翻译计划”主要处理知识表达中的语言污染。

二者关系可以理解为：

- **知识地基计划**：问“这个知识站在哪些前提之上？”
- **清朗翻译计划**：问“这个知识为什么被讲得这么难懂？”

一个挖地基，一个清管道。

前者让知识更可审查，后者让知识更可理解。

## 6. 当前定位

本计划目前只是一个项目种子，不抢当前主线。

短期目标是保存问题意识，后续可在合适时机选择一个小领域做最小样例，例如：

- 脑科学术语清朗翻译
- 心理学术语反黑话
- AI 概念工程类比词典
- 物理学入门术语纠偏

核心原则保持不变：

> 不是替代专业知识，而是清除不必要的认知污染；不是让人不学习，而是让学习者把精力用在真正的知识结构上。
